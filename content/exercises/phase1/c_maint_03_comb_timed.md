# ft_print_comb timed

## Завдання

Напиши функцію `ft_print_comb` **з пам'яті** під таймером 15 хвилин.

Це тренування **швидкості під тиском**. На справжньому examshell тебе ніхто не чекатиме. Кожна хвилина, витрачена на просту функцію, -- це хвилина, яку ти не витратиш на складне завдання, що приносить більше балів.

`ft_print_comb` -- це C00 ex05 Piscine. Вона складніша за `ft_putstr` чи `ft_putnbr`, бо потрібні три вкладені цикли та правильне форматування виводу. Але після Phase 0 ти вже це робив.

### Інструкція з таймером

1. **Постав таймер на 15 хвилин** (телефон, `timer 15m`, що завгодно)
2. Відкрий порожній файл `ft_print_comb.c`
3. Пиши код. НЕ відкривай жодних матеріалів
4. Коли таймер пропищить -- **зупинись, навіть якщо не закінчив**
5. Скомпілюй та запусти тести
6. Запиши свій час та результат

### Прототип

```c
void	ft_print_comb(void);
```

### Вимоги

- Напиши `ft_print_comb` з пам'яті за 15 хвилин
- НЕ відкривай рішення з Phase 0, НЕ дивись в інтернет
- Виведи всі комбінації трьох різних цифр (0-9) у порядку зростання
- Цифри у комбінації також у порядку зростання: `012`, НЕ `021`
- Роздільник: `, ` (кома + пробіл) між комбінаціями
- Після останньої комбінації (`789`) -- нічого
- Використовуй тільки `ft_putchar`
- Тільки цикл `while` (НЕ `for`)
- Файли: `ft_print_comb.c` + `ft_putchar.c`
- Norminette: так

### Очікуваний результат (скорочено)

```
012, 013, 014, 015, 016, 017, 018, 019, 023, 024, ..., 789
```

Повний вивід: 120 комбінацій, 119 роздільників.

### Тестування

```c
void	ft_print_comb(void);

int	main(void)
{
	ft_print_comb();
	return (0);
}
```

### Таймер

| Час | Результат |
|-----|-----------|
| < 10 хв | Бездоганно -- ти пишеш як досвідчений piscineux |
| 10-15 хв | Добре -- вписався в ліміт |
| > 15 хв | Потрібна додаткова практика. Повтори завтра |
| Не скомпілювався | Повтори Phase 0 ft_print_comb, поверни навичку |

## Підказки

<details>
<summary>Мета-підказка: стратегія швидкого написання</summary>

На екзамені найефективніший підхід:

1. **Спочатку структура** -- напиши всі три вкладені `while`, порожні, з правильними межами
2. **Потім вивід** -- додай `ft_putchar` для трьох цифр
3. **Потім роздільник** -- `if` для перевірки "не останній"
4. **Потім компіляція** -- `-Wall -Wextra -Werror`

Не намагайся написати все одразу. Структура -> вивід -> деталі -> тест.

**Це єдина підказка.** Технічних деталей немає -- ти вже все знаєш.

</details>
