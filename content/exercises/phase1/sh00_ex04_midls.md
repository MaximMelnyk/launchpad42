# midLS

## Завдання

Запиши у файл `midLS` одну команду `ls` з правильними опціями, яка виводить вміст поточної директорії у форматі:

- Довгий формат (long listing)
- Розділення комами
- Сортування за часом модифікації (новіші спочатку)
- Показувати `/` після директорій

Ця вправа навчить тебе опціям команди `ls` -- однієї з найчастіших команд у терміналі. На Piscine ти будеш постійно використовувати `ls` для перевірки своїх файлів перед здачею.

### Вимоги

- Файл `midLS` повинен містити **одну команду** (один рядок)
- Команда повинна використовувати `ls` з опціями: `-l`, `-m`, `-p`, `-t`
- **Порядок опцій** може бути скомбінований: `ls -lmpt` або `ls -l -m -p -t`
- Файл закінчується символом `\n`
- Здай у директорії `ex04/`

### Опції

| Опція | Значення |
|-------|----------|
| `-l` | Long format (детальний вивід з правами, розміром, датою) |
| `-m` | Розділення комами (для файлів у рядку) |
| `-p` | Додає `/` після імен директорій |
| `-t` | Сортування за часом модифікації (newest first) |

### Перевірка

```bash
$ cat midLS
ls -lmpt
$ bash midLS
total 8
drwxr-xr-x 2 user group 4096 Jun  1 23:42 dir1/,
-rw-r--r-- 1 user group   42 Jun  1 23:00 file.txt
```

Вивід залежить від вмісту поточної директорії -- це нормально.

## Підказки

<details>
<summary>Підказка 1: Створення файлу з командою</summary>

```bash
echo "ls -lmpt" > midLS
```

Або створи файл у Vim:
```bash
vim midLS
# Натисни i, введи: ls -lmpt
# Натисни Esc, потім :wq
```

</details>

<details>
<summary>Підказка 2: Тестування команди</summary>

Спочатку спробуй кожну опцію окремо, щоб зрозуміти ефект:

```bash
ls -l     # Довгий формат
ls -m     # Через коми
ls -p     # Слеш після директорій
ls -t     # Сортування за часом
ls -lmpt  # Всі разом
```

</details>

<details>
<summary>Підказка 3: Перевірка вмісту файлу</summary>

```bash
# Переконайся, що файл містить рівно один рядок
wc -l midLS
# Очікувано: 1

# Перевір, що \n в кінці
cat -e midLS
# Має закінчуватись на $
```

</details>
