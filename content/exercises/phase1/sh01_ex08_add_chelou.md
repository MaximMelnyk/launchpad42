# add_chelou

## Завдання

Напиши командний рядок, який додає два числа, записані у "дивних" (chelou) системах числення, і виводить результат у третій "дивній" системі.

"Chelou" -- це французький сленг, що означає "дивний, незвичний" (від арабського "khalouf", перевернуте "louche"). Назва натякає на те, що числа записані не цифрами, а спеціальними символами.

Це найскладніша вправа Shell01. Вона вимагає розуміння систем числення та вміння конвертувати між ними за допомогою shell-інструментів.

### Файли для здачі

- Директорія: `ex08/`
- Файл: `add_chelou.sh`

### Системи числення

**FT_NBR1** -- число у системі з основою 5, де цифри замінені символами:

| Символ | Значення |
|--------|----------|
| `'` (апостроф) | 0 |
| `\` (зворотний слеш) | 1 |
| `"` (лапки) | 2 |
| `?` (знак питання) | 3 |
| `!` (знак оклику) | 4 |

**FT_NBR2** -- число у системі з основою 5, де цифри замінені літерами:

| Символ | Значення |
|--------|----------|
| `m` | 0 |
| `r` | 1 |
| `d` | 2 |
| `o` | 3 |
| `c` | 4 |

**Результат** -- сума у системі з основою 13, де цифри замінені:

| Символ | Значення |
|--------|----------|
| `g` | 0 |
| `t` | 1 |
| `a` | 2 |
| `i` | 3 |
| `o` | 4 |
| ` ` (пробіл) | 5 |
| `l` | 6 |
| `u` | 7 |
| `S` | 8 |
| `n` | 9 |
| `e` | 10 (A) |
| `m` | 11 (B) |
| `f` | 12 (C) |

### Вимоги

- Скрипт має починатися з `#!/bin/sh`
- Числа беруться зі змінних `$FT_NBR1` та `$FT_NBR2`
- Сума виводиться у системі з основою 13

### Приклад

```bash
$ export FT_NBR1="\\\\'"
$ export FT_NBR2="dcm"
$ bash add_chelou.sh
ai
```

Пояснення:
- `FT_NBR1` = `\\'` = 1, 1, 0 (в 5-ковій) = 1*25 + 1*5 + 0 = 30
- `FT_NBR2` = `dcm` = 2, 4, 0 (в 5-ковій) = 2*25 + 4*5 + 0 = 70
- Сума = 100 (в десятковій)
- 100 в 13-ковій = 7*13 + 9 = 79 -> ні, 100 = 7*13 + 9 -> "u" (7) + "n" (9) = `un`

Зачекай, перерахуємо: 100 / 13 = 7 (залишок 9). Отже 100 в base 13 = 79. Символи: 7=`u`, 9=`n`. Результат: `un`.

### Очікуваний формат виводу

Результат додавання, записаний символами системи з основою 13.

## Підказки

<details>
<summary>Підказка 1: стратегія</summary>

Алгоритм:
1. Замінити символи `FT_NBR1` на цифри 0-4 (за таблицею)
2. Замінити символи `FT_NBR2` на цифри 0-4 (за таблицею)
3. Скласти два числа у системі 5 і перевести в систему 13 (за допомогою `bc`)
4. Замінити цифри результату на символи виводу

Для конвертації між системами числення використай `bc`:
```bash
echo "ibase=5; obase=..." | bc
```

</details>

<details>
<summary>Підказка 2: sed для заміни символів</summary>

Використай `sed` для заміни кожного символу на відповідну цифру:

```bash
# FT_NBR1: ' -> 0, \ -> 1, " -> 2, ? -> 3, ! -> 4
echo $FT_NBR1 | sed "s/'/0/g" | sed 's/\\/1/g' | sed 's/"/2/g' | sed 's/?/3/g' | sed 's/!/4/g'
```

Для `FT_NBR2` використай `tr`:
```bash
echo $FT_NBR2 | tr "mrdoc" "01234"
```

</details>

<details>
<summary>Підказка 3: bc для конвертації баз</summary>

`bc` -- це калькулятор, який підтримує різні бази числення:

```bash
echo "ibase=5; obase=23; 1234" | bc
```

Увага: `obase` задається в одиницях `ibase`! Якщо `ibase=5`, то `obase=23` означає 2*5+3=13 (тринадцять) у десятковій.

Результат `bc` у базі > 10 виводить цифри як двозначні десяткові числа: `00`, `01`, ..., `12`.

Потім замінюєш їх на символи виводу за допомогою `tr`:
```bash
... | tr "0123456789ABC" "gtaio luSnemf"
```

</details>
